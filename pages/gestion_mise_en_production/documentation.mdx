# Documentation

Lorsque les installations ont été validées, le déploiement approche de sa conclusion. Cependant, l'exploitation requiert des informations détaillées sur les éléments déployés, y compris leurs versions et les interdépendances entre les composants.

Il est courant que les nouveaux éléments soient enregistrés dans la base de données avant même le déploiement, avec un statut indiquant qu'ils ne sont pas encore opérationnels.

La fin du déploiement est l'occasion idéale pour finaliser la documentation d'exploitation, qui doit inclure :

- Les spécifications des éléments ajoutés ou modifiés, remplaçant les anciennes valeurs le cas échéant.
- Les manuels d'administration de la nouvelle solution ou version.
- L'état des paramètres de configuration.

Si les éléments ont déjà été saisis dans la CMDB (Configuration Management Database), la conclusion du déploiement permet de mettre à jour le statut des Configuration Items (CI), faisant passer leur statut de "en projet" ou "en cours" à "actif" ou "en production".
